JavaScript runtimes contain a message row that stores a list of messages to process and their associated callback functions. These messages are grouped in response to external events (such as clicking a mouse or responding to an HTTP request) A callback function is provided. For example, if a user clicked a button and no callback function was provided - no message would be queued. In a loop, the row is polled for the next message (each poll is referred to as a "tick") and when a message is encountered, a callback is executed for that message.
